# 简历项目描述

## 推荐项目标题

电商零售销售与用户行为分析项目｜SQL + Python + RFM + 用户分层

## 三行版

- 基于模拟电商零售订单数据，使用 Python 构建数据生成、清洗与分析流程，并通过 SQLite 完成订单、用户、品类等交易数据的结构化存储与查询。
- 使用 SQL 构建 GMV、订单量、活跃用户、客单价、复购率等核心经营指标，按月份输出经营趋势，并识别收入波动和用户复购变化。
- 基于 Recency、Frequency、Monetary 构建 RFM 用户分层模型，将客户划分为高价值、忠诚、潜力、流失风险、沉睡等群体，并针对不同客群提出会员运营、优惠券召回和交叉销售策略。

## 强化版

- 独立完成电商零售销售与用户行为分析项目，使用 Python 生成并清洗 5,000+ 条模拟订单数据，构建客户、订单、品类等分析表，并通过 SQLite 建立可复用分析流程。
- 使用 SQL 的 JOIN、CTE、CASE WHEN、聚合函数完成月度 GMV、订单量、活跃用户、客单价、复购率等指标计算，输出 `monthly_kpi.csv` 支持后续 Excel/BI 看板展示。
- 基于 RFM 模型构建客户价值分层，结合最近购买时间、购买频次和消费金额识别 Champions、Loyal Customers、At Risk、Hibernating 等用户群体，输出客户级分层表和分层汇总表。
- 根据分层结果提出精细化运营建议：对高价值用户设计会员权益与推荐激励，对潜力用户推送品类推荐和门槛券，对流失风险用户设计限时召回活动。

## 面试讲法

这个项目主要模拟了电商业务中从交易数据到运营策略的完整分析过程。我先用 Python 生成客户和订单数据，再用 SQLite 做结构化查询。分析部分分两层：第一层是经营指标，比如 GMV、订单量、活跃用户、客单价和复购率；第二层是客户价值分层，用 RFM 模型把用户分成高价值、忠诚、潜力、流失风险和沉睡用户。最后我把结果导出成 CSV，可以接 Excel、Power BI、Tableau 或 Streamlit 做看板。
